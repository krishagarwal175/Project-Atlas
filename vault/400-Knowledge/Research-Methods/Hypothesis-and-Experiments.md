# 🧰 Hypotheses & Experiments

> A belief you can't disprove isn't knowledge, it's faith. A good experiment states a falsifiable hypothesis, defines success/failure *before* running, and produces a signal that actually changes what you do.

## What it is
The scientific method, applied to strategy. A hypothesis: "If we do X, then metric Y moves by Z within time T." An experiment: the smallest test that could return *refuted*. The discipline is pre-registration — decide the failure condition and the persevere/pivot rule before you see data, so you can't rationalize afterward ([[MM-cognitive-biases|confirmation bias]]).

## When to use it
Every strategic bet with uncertainty. This is the engine behind Atlas's [[TPL-experiment|Experiment Planner]] and [[F-lean-startup]].

## How to run it (quality checklist)
1. **Falsifiable** — could it come back false? 2. **Leading indicator** — an early signal, not a lagging one. 3. **Failure condition armed** — the kill line stated up front. 4. **Smallest sufficient test** — minimize cost-to-signal. 5. **Pre-committed interpretation** — what each result means, decided now.

## Why it works
Pre-registration and falsifiability defeat the biases that make us "learn" whatever we hoped. It converts opinion into evidence the [[Vault-Governance|governance]]/epistemics engine can weigh.

## Limitations / when NOT to use it
- Not everything is cheaply testable; some bets need judgment + [[MM-scenario-planning]].
- Underpowered tests (too little data) return noise dressed as signal.
- Metrics can be gamed ([[F-okrs-north-star]] Goodhart).
